Arkitektur

Slik er hvert workspace strukturert i GitHub, og hvordan kode flyter fra utvikling til produksjon.

Branch-strategi

dev
Utviklingsmiljø — jobb her, push fritt
Pull Request
Review + godkjenning kreves
main
Produksjon — auto-deploy til Fabric

Repo-struktur

  • workspace.json — metadata
  • rapport.pbix — Power BI Desktop-fil
  • model/ — TMDL for code review
  • dataflows/ — JSON-eksport
  • .github/workflows/ — CI/CD

For Power BI-utviklere

  • Klon repo, sjekk ut dev
  • Åpne .pbix i Power BI Desktop
  • Lagre, commit og push til dev
  • Lag PR til main for release
  • GitHub Actions deployer automatisk

CI/CD

  • Push til dev → deploy til dev-workspace
  • Merge til main → deploy til prod
  • Fabric REST API via Service Principal
  • Katalogen oppdateres automatisk

Nytt workspace

  • Fork workspace-template-repo
  • Fyll ut workspace.json
  • Legg til i repos.txt i katalogen
  • Sett Fabric workspace ID i Secrets
  • Ferdig — klar for utvikling
