Arkitektur

Slik er hvert workspace strukturert i GitHub og Fabric.

🗂️ GitHub-struktur

  • main → Produksjonsmiljø
  • dev → Utviklingsmiljø
  • Pull request med review
  • CI/CD via GitHub Actions
  • Auto-deploy til Fabric

📊 Fabric-innhold

  • Power BI Desktop (.pbix)
  • Semantic models (.bim / TMDL)
  • Dataflows (JSON)
  • Notebooks (.ipynb)
  • Pipelines (JSON)

🔄 Workflow

  • Utvikler jobber i dev
  • Åpne i Power BI Desktop via repo
  • PR til main for release
  • Actions deployer til Fabric-workspace
  • Katalogen oppdateres automatisk

🛡️ Tilgangsstyring

  • GitHub Teams per workspace
  • Branch-protection på main
  • Fabric: Viewer / Contributor / Admin
  • Service Principal for CI/CD
  • Sensitivity labels (MIP)

⚡ Komme i gang

Bruk workspace-templaten for å spinne opp et nytt workspace på 2 minutter.

  • Fork workspace-template
  • Fyll ut workspace.json
  • Kjør setup-action
  • Ferdig — klar for utvikling

📡 Denne katalogen

GitHub Pages-side som leses fra workspace.json i hvert repo.

  • Automatisk oppdatering via Actions
  • Ingen backend — kun statisk HTML
  • Åpen for alle med GitHub-tilgang
  • Direkte lenke til Fabric + Desktop
