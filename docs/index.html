Slik fungerer det

Her er en oversikt over hvordan workspaces, rapporter og semantiske modeller henger sammen — og hva som skjer bak kulissene når du gjør en endring.

Hvert workspace er ett område

Salg, Finans, Verksted osv. er egne workspaces i Fabric. Alle rapporter og modeller for et område ligger samlet der.

Rapporter (.pbix)
Semantiske modeller (TMDL)
Dataflows og notebooks

To versjoner alltid

Hvert workspace finnes i to varianter automatisk. Du jobber alltid i arbeidsmiljøet — aldri direkte i det som er live.

Produksjon — det brukerne ser
Arbeidsmiljø — her tester du

Rapporter — redigeres i Desktop

Power BI-rapporter lastes ned som .pbix, redigeres i Power BI Desktop, og lastes opp igjen her. Endringen kjøres gjennom automatisk gjennomgang før den publiseres.

Last ned → Rediger i Desktop → Last opp → Gjennomgang → Publisert

Semantiske modeller — redigeres via kobling

Semantiske modeller inneholder datadefinisjonene som rapportene bygger på. De kan ikke lastes ned som en fil — i stedet kobler du deg direkte til modellen via Tabular Editor eller Power BI Desktop.

Tabular Editor — full redigering av DAX og relasjoner
Power BI Desktop — live-kobling via XMLA
Endringer lagres direkte i arbeidsmiljøet

Automatisk gjennomgang

Når du laster opp en endring sjekkes den automatisk for vanlige feil og beste praksis — DAX-mønstre, skjulte kolonner, manglende beskrivelser og mer. Du velger selv om du vil rette opp eller sende inn uansett.

Advarsler — bør fikses
Informasjon — greit å vite
OK — ser bra ut

Semantiske modeller kan ikke lastes ned som en fil. De redigeres ved å koble seg direkte til arbeidsmiljøet via ett av verktøyene under.
Tabular Editor
Anbefalt for DAX-utvikling, relasjoner og modellstruktur. Kobler seg til arbeidsmiljøet via XMLA-endepunkt og gir full kontroll over modellen.
Power BI Desktop — live-kobling
Åpne Desktop og koble til arbeidsmiljøet via «Hent data → Power BI-datasett». Du kan da bygge rapporter på toppen av modellen uten å laste ned noe.
